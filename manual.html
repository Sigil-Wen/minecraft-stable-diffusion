Minecraft Map Art Maker

Upload an image to create an Add-On (Bedrock edition 1.17) that builds the image in your world, to view it on a map!

Browser Compatibility

It is recommended to use this app on a desktop/laptop instead of on mobile!

This is a client-side app, i.e. it functions entirely in your browser (No data is stored in or sent to an external server for processing !). Hence, some essential functions may not work if your browser hasn't been updated to the latest/modern versions, or your device places many restrictions on it.
You must have Javascript enabled for this site. In additon, it is recommended that atleast 10 MB memory is available for this site (or don't upload too many images into one pack). This site does not use any cookies or local storage.


Using This Web App

Some notes about the generated Behaviour Packs :

  • When you create the add-on, commands to build all the images that were uploaded and processed will be written into it. Even a single map contains 128×128 = 16384 visible blocks to be placed, which is a lot !, so the commands will be split into 2 sets per map and written into seperate function files (they are regular commands like /setblock, but Minecraft can then execute upto 10,000 of them at once from these files). More instructions on running the functions are given below.
    Read more about functions in Minecraft.

  • The commands use relative ˜ (tilde notation) to specify the positon, so that the image will be built wherever you (the player, or a command block if you are using one) are standing, instead of a specific place in each world every time.

  • Common Starting Coordinates : Normally, the coordinates of any block in each function (64×128 region, half of a map) are specified relative to its own top-left (North-West corner). After executing the function, you would need to be at the next unbuilt map half before running the next function there.
    However, if this option was checked, the coordinates of every block in the whole map art will be relative to the corner of the first function only (coincident with the NW corner of the whole image) - so you can stand there and run all the commands without moving.
    This was not recommended just in case (a) blocks thousands of meters away, beyond simulation distance may not get placed, and (b) if there is another player nearby, they may interfere with the command placement.
  • Replace only air : If this is enabled, the volume (rectangle for a flat art, cuboid for 3D) in which the blocks will be placed will not be cleared / filled with air before placing the blocks. In addition, blocks that need to be placed at locations where there is an existing non-air block will not replace it.
    Note: The Glow lichen uses a cobblestone block below it for support, it will also not be placed if the cobblestone can't.



In Minecraft

  1. Import and apply the behaviour pack to your world(s).
    Note : Cheats & Operator permissions must be enabled to use functions. If you want to earn achievements in that world or keep cheats off, you'll have to place the blocks manually.
  2. Go to the location where the image is to be built, and open a Level 0 (smallest size) map there. Check that the starting block you choose is aligned with the top-left (NW) corner of the map !
  3. This block will be the top left (North-West) corner of the entire image (see the green marker in the figure below), and the image will extend anywhere from 128 - 512 blocks (1-4 maps) South and East from here, depending on the size you've chosen. Height-wise, it will be built at the block in which you are standing, for the 2D, or ranging from that y-level till (specified max height) blocks above it, for the 3D palette.*
  4. If your image has a size of axb maps for example, there will be 2 × a × b functions to be executed individually, called function_name/1, function_name/2, ... so on. Each of these builds a part of the whole image, covering a vertical half of one map, arranged in order from West to East, then North to South. The figure to the right shows an example.
    • If 'Use same starting coordinate' is enabled, run all the functions from this starting corner only (top-left of the whole image, marked with a small green square in the figure).
    • If this option was not enabled, you will need to be at the next zone before building it - marked by small black squares in the figure. You won't have to move manually, in this case all the functions up till the last will also contain /tp commands to automatically teleport you to the next corner, and also try to place a Cobblestone block there to identify it exactly.
    Note : You can even run the functions from command blocks placed at those locations instead of standing at the corner and entering the command yourself.
  5. The syntax of the function command is /function image_name/number.
    For example, if your image's function name was wallpaper, and its size was 2x1, you would need to run the commands 
    /function wallpaper/1
/function wallpaper/2
/function wallpaper/3
/function wallpaper/4
  6. You're Done !

    Once you've constructed the image, map the entire area covered by it and build a map wall for your pixel art !
    It's also a good idea to lock the maps with glass panes to prevent them from updating if some other structures are built above them, or if the colour palette changes in future updates.

* For the 3D arrangement, the max height specification only ensures that blocks do not get placed beyond that height, but the limit may not be reached at all - it depends on the colours in the image. You may find, after building it, that the blocks did not reach that height anyway.




Example behaviour pack

This is the file structure of an example behaviour pack generated by this app. Assume it was created using 2 images, one with the function name "cliff" and of size 2x1, and the other with the function name "bastion" and of map size 1x1. Directory names are written with an ending / and the rest are files :


  behaviour pack.mcpack
   |
   |-- manifest.json
   |-- pack_icon.png
   ├-- structures/
   |    └ mapart/
   |       ...
   └-- functions/
        |
        ├-- cliff/
        |    |-- 1.mcfunction
        |    |-- 2.mcfunction
        |    |-- 3.mcfunction
        |    └-- 4.mcfunction
        |
        └-- bastion/
             |-- 1.mcfunction
             └-- 2.mcfunction

Download a sample pack

  • The manifest specifies :
    • The minimum game version - 1.17.0 (of Minecraft)
    • Behaviour pack version - 1.0.0 is used for each pack by default
    • The name and description that you entered, will be displayed for the add-on in game.
    • 2 version-4 UUIDs to uniquely identify the pack, generated securely by an external service. If their server does not respond, you will be prompted to enter 2 UUIDs manually - any UUID (v4 or other) is sufficient as long as it has not been reused or copied from another existing pack that you are using / may use.
    You can always edit the icon or manifest if you prefer to make any changes later on.

To view/edit/replace the files inside, just rename the .mcpack extension to .zip and extract all contents using any Compressed (ZIP) folder utility.
