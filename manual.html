Minecraft Map Art Maker

Upload an image to create an Add-On (Bedrock edition 1.16) that builds the image in your world, to view it on a map!

Manual

Browser Compatibility

It is recommended to use this app on a desktop/laptop instead of on mobile!

This app functions entirely in your browser (No data is ever stored in or sent to any external server!). While this is advantageous for speed and security, the trade-off is that some essential functions may not work if you haven't updated your browser software to the latest version in a while, or your device places many restrictions on it.
You must have Javascript enabled for this site. In additon, it is recommended that atleast 10 MB memory is available for this site (or don't upload too many images into one pack). This site does not use any cookies or local storage.

Using This Web App

Minecraft Map Art Maker lets you upload images and select the colour palette and size of the pixel art to be created from each one, then processes them to create new images that resemble what can be shown on Minecraft maps (in Bedrock edition) as closely as possible. You can then generate a behaviour pack containing commands that will place blocks to build the images in your world!

Follow the steps below :

The points with this symbol contain technical details that you can ignore if you're not interested.

  1. Scroll to the  Upload Images  section, in which there is a form titled 'Image 1'. Using the file chooser, upload an image from your system (any image type - JPG/JPEG, PNG, GIF, TIFF, etc that is recognised by your browser as an image can be uploaded). Avoid choosing large images (with a size larger than around 1000px, or 1-2 MB) as this may fill up too much of memory available to your browser and also since the resized/pixelated version (â‰¤ 512px) may not look very similar to the original.
  2. Choose an area (as width x height) that the image built in your world should cover. These numbers (ranging from 1x1 to 4x4) specify the number of Level 0 (zero) maps (The smallest map size in minecraft, displaying an area of 128x128 blocks, or, 1 pixel for each block) to cover.
  3. Choose a colour palette from 3 options - Basic, Standard or Extended. Basic and Standard use flat / horizontal arrangements of blocks only; while Extended is 3 dimensional, extending upto a maximum height that you must specify View the colours used for these palettes below.
  4. The Function name is the identifier that will be used to refer to that image in the commands (it can only contain the characters A-Z a-z 0-9 _ - . ( ) due to Minecraft command syntax)
  5. Depending on the colour palette chosen, you might also have to fill in an extra option of the Building material (for Basic only) or Maximum height (for Extended only)
  6. Dither is an image processing effect that controls how accurately any region of the converted image has similar colour/appearance to the original image, by introducing noise/grain of different colours as required - try toggling this On/Off for a different look !

  7. Click on the  Process Image  button to finish resizing, pixelating and converting its colour palette. After this, the form will be disabled so that the settings for this image can't be changed again. Three new buttons -  Original  ,   Resized  &   Converted   will be added. Click on these to view the corresponding images.
    The resized/converted images may actually be very small in size (128 to 512 px), so they are displayed enlarged. Right click on the previewed image and open it in a New Tab to see the actual size, or also copy / save these images if you would like to.
  8. You can process as many images as you like to write into the same behaviour pack - however it is not recommended to add more than 8-10 as this can cause problems downloading the large file produced (~1 MB per small image). To add a new image, click on the icon to the top right of the image form area, and a new tab will be created. Follow the same guidelines as above while filling it in. A new tab is added each time, and its title changes to the function name of its image, provided by you, after processing.
  9. If you've processed an image but don't want to use it anymore, you can remove it permanently by clicking the  Delete  button in the bottom right of the processed image form. Warning : This action can't be undone ! You can delete all the images if you like and still continue to write the behaviour pack - this will just result in a 'blank' pack with no content other than its ID and icon being generated.

To create the Behaviour pack :

  1. Using an add-on, you can bundle commands to build all the uploaded & processed images into a single file. Navigate to the  Create the Add-On  section below the image forms and enter a name and description for the behaviour pack. These will be displayed in the Behaviour packs / Storage menu in-game, after the pack has been imported.
  2. The behaviour pack will consist of a manifest.json file, a pack_icon.png and a functions folder at the toplevel. The manifest contains the name, description, and other data required by minecraft such as :
    • The minimum game version - 1.16.0 is used
    • Behaviour pack version - 1.0.0 is used for each pack by default
    • 2 UUIDs to uniquely identify the pack - version 4 UUIDs generated securely by an external service are used by default, but if their server does not respond, you will be prompted to enter 2 UUIDs manually - any UUID (v4 or other) is sufficient as long as it has not been reused or copied from another existing pack that you are using / may use. The pack_icon generated by default contains snippets of the first 4 images that have been processed (or of all the images, if less than 4 were processed).
    You can edit the icon or manifest if you want to make any changes later on.
    See an example behaviour pack structure below.
  3. The functions folder will contain a subfolder for each processed image, named its image's Function Name. Each of these folders may contain 2 - 32 files with commands, depending on the image size.
    This is because Minecraft places a recursive upper limit of upto 10,000 commands being executed from any function call, and even the smallest 1x1 image size covers an area of 128×128 = 16,384 blocks, requiring at least that many /setblock commands. Hence, each map area is split into 2 halves and one function is written for each zone.
    The /setblock commands use relative ˜ (tilde notation) to specify the positon, so that the image will be built wherever you (the player, or a command block if you are using one) are standing, instead of a specific place in each world every time.

  4. Common Function Coordinates : If this is enabled, the coordinates of every block in the image, across all functions, will be written with reference to the same starting point (  ~ ~ ~   is the top-left (NW) corner of the image, also the NW corner of the first zone) - so you should run all functions from that same location. Otherwise, the reference for each function is the top left (NW) corner of its zone only, and you have to move to the corner of the next zone each time before filling in the blocks there.
    It is recommended not to enable this and move manually instead (depending on the image size and your simulation distance ) since for large images blocks far away from they player, outside the sim distance or any ticking area, will not be placed, leaving odd gaps in the picture.
  5. Replace only air instead of destroying : If this is enabled, wherever there are any non-air / non-empty blocks at a location where a block for the image has to be placed, those blocks are not replaced but are preserved (this will leave a "gap" or a differently coloured spot in the image at those places, if there are any). Note : When this is disabled (normally), the extended / 3D arrangement clears the entire cuboidal volume from the starting top-left-lowest corner to the bottom-right-highest (including spaces in which there are no blocks, above/below each image block). If enabled, this volume is not cleared.
  6. Click on the  Write Functions  button at the end of the form to analyse all the processed images and write commands for them, and combine everything into a ZIP file. When this process is complete, the button will be replaced with a   Download   button. Click on it to download the pack, named pixelart.mcpack by default - you can rename this to whatever you like.

    Internally, the file before downloading is stored as a Blob file - this format may not be fully supported by some older browsers, so in case it doesn't work or the download is incomplete or missing some data, you can try the alternate link provided below it, which stores it temporarily as a Data URI (more widely supported, although this has much smaller size limits, so it may not work too if you have uploaded more than 1-2 images or have enabled memory usage restrictions in your browser. There is no alternative to these).



In Minecraft

  1. Import and apply the behaviour pack to your world(s).
    Note : Cheats must be enabled to use functions. If you want to earn achievements in that world or keep cheats off, you'll have to place all the blocks manually - you can refer to the processed images or the function files from the pack as a guide.
  2. Go to the place in your world where you want to build the image, and open a Level 0 (starting size) map there. If you want the image to be aligned exactly with the edges of the map (no gaps/edges), go to the top-left corner of that map and mark that as the block from which to start building the image instead.
  3. This will be the top left corner of the image (see the green square in the figure below), and the image will extend anywhere from 128 - 512 blocks (1-4 maps) South and East from here, depending on the size you've chosen. Height-wise, it will be built at the block in which you are standing, for the flat (basic / standard) palettes, or ranging from that block till (specified max height) blocks above where you are standing, for the 3D (extended) palette. *
  4. If your image has a size of axb maps for example, there will be 2 × a × b functions to be executed individually, called function_name/1, function_name/2, ... so on. Each of these builds a part of the whole image, covering a vertical half of one map (64×128 blocks). These are arranged in order from West to East, then North to South. The figure to the right shows an example for a 2x2 image.
    • If 'Use same starting coordinate for all functions' is enabled, run all the functions from this corner only (top-left of the whole image, marked with a small green square in the figure).
    • If this option was not enabled, you need to be at the top-left corner of each zone before running the function for that zone - marked by small black squares in the figure. You won't have to move manually, as in this case all the functions up till the last will also contain /tp commands to teleport you to the next corner, and also try to place a Cobblestone block there to identify it exactly.
    Note : You can even run the functions from command blocks placed at those locations instead of standing at the corner and entering the command yourself.
  5. The syntax of the function command is /function image_name/number.
    For example, if your image's function name was wallpaper, and its size was 2x1, you would need to run the commands 
    /function wallpaper/1
/function wallpaper/2
/function wallpaper/3
/function wallpaper/4
    Read more about functions in Minecraft.
  6. You're Done !

    Once you've constructed the image, map the entire area covered by it and build a map wall for your pixel art !
    You can also optionally lock the maps with glass panes to prevent them from updating even if some other structures are built above your image blocks, covering them.

    If you also want to rotate sections of the image, you can copy and rotate them in your world using Structure blocks.

* For the 3D arrangement, the max height specification only ensures that blocks do not get placed beyond that height, but the limit may not be reached at all - it depends on the colours in the image. You may find, after building it, that the blocks did not reach that height anyway.
